# Customer Module

## Owns

The customer module owns customer identity and profile information.

It answers questions like:

- does this customer exist?
- who is this customer?
- what records belong to this customer?

## Examples

- customer name
- customer email
- customer profile
- customer ownership checks

## Does Not Own

- credit limit
- card balance
- transaction approval
- ledger events
- billing cycles
- notifications
- analytics

## Rule

Customer owns identity.

Accounts/cards own credit usage.
