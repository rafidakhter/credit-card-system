# Audit Module

## Owns

The audit module owns records that explain **who did what and when**.

It helps us understand the actions that happened in the system.

## Examples

- purchase approved
- purchase declined
- transaction status changed
- refund requested
- refund completed

## Does Not Own

- money movement
- customer balances
- transaction state rules
- billing calculations
- notifications

## Rule

Audit records explain actions.

Ledger records explain money.
