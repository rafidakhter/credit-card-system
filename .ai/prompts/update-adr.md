# Prompt: Update an ADR

Use this when a decision changes the system architecture or long-term direction.

## Instruction

Create or update an ADR for one architectural decision.

Use this structure:

```text
# ADR-NNNN: Title

## Status

Proposed | Accepted | Superseded

## Context

What problem forced this decision?

## Decision

What are we choosing?

## Consequences

What improves?
What gets harder?
What should future agents not accidentally undo?
```

## Constraints

- ADRs are for decisions, not implementation notes.
- Keep the language plain.
- Include trade-offs.
