# Task 004: Design Initial Domain Services and Repository Contracts

## Goal

Define the first domain services before writing more backend behavior.

## Scope

- Define the first core services for Phase 1.
- Define what each service is responsible for.
- Define the repository contracts those services need.
- Keep database writes behind service logic instead of designing from seed data first.
- Decide where minimal local seed data still helps development later.

## Acceptance Criteria

- The project has a clear first-pass service list.
- Each service has a simple responsibility.
- Repository contracts are described in plain language.
- The task notes make it clear that the domain drives persistence, not the other way around.
